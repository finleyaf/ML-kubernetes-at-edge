\textbf{Context}

• Container orchestration is achieved through Kubernetes with automated deployment, scaling, and error handling.

\textbf{Problem}
\textit{why resource management in Kubernetes scheduling is not good}
• Orchestration exposes the need for efficient scheduling.
• Essential in mission-critical applications such as healthcare, telecommunications, and forecasting.
• Pod placement involves selecting hardware, location, resource needs, and load balancing in the assignment of pods to nodes.
• The Kubernetes scheduler is primarily designed for homogeneous data centers with predictable resources, unsuited to heterogeneous edge environments that require dynamic resource allocation.
• Monitoring such an unreliable environment is difficult.
\textit{more detail is needed}

\textbf{Existing approaches}
• Static schedulers are just one example of an ineffective solution for edge environments.
• Monitor-driven heuristics attempt to improve scheduling by directly reacting to changes in resource consumption.
• Monitoring acting solely on past states means it struggles to get ahead in transient environments like the edge, specifically burst-like workloads.
• Plus overly sensitive to noise.
• Learning models for anomaly detection are used to monitor resources and recognise deviations from normal behaviour.
• Poor due to binary nature and sensitivity to preprocessing.

\textbf{The gap}
• After showing that existing approaches either lack adaptability, overreact to noise, or impose excessive computational overhead for edge environments.
• The motivation for lightweight models that combine detection and prediction becomes clear.

\subsection{Edge computing fundamentals}
\textit{Why is anomaly detection necessary?}
• Anomaly detection involves monitoring nodes to achieve a successful deployment.
\textit{Motivation behind the choice of models for anomaly detection.}

• The demand for data collection and analysis in all possible ways lead to the promise of a cloud computing infrastructure, using centralised remote servers to store and process all data - reducing the need for physical infrasture directly supports network scaling.
• Introducing IoT devices into the network, devices which collect and send data in real-time, means massive amounts of unused data is being transfered across the network with no use case.
• Edge computing scales with increasing amount of data by moving computation as close as possible to collection while limiting long distance data transfer to only when necessary. This also allows for IoT devices to make intelligent decisions on what data should be sent across the network - resulting in fewer burst-like transfers.
• Edge network are made up of devices, servers and gateways which are labelled as nodes. Since many of the devices are small mobile devices and sensors, naturally the resources available for processing are constrained. Also the distributed nature spans large areas geographically which poses challenges in connectivity.
• The larger the network the more diverse the nodes appear to be. Edge networks need capabilities to support heterogeneous hardware.

\subsection{Machine learning in resource management}

...Anomaly detection techniques for resource monitoring
• The statistical methods relevant to anomaly detection in my experiment are threshold and z-score. Threshold is used when measuring metrics and the target resulting should lie within a set range of values. Then the metrics can be measured simply as a binary result of within or not within the range, or even distance from the range. Z-score is a threshold method measures the number of standard deviations from the mean for binary classification. Adjusting the z-score threshold allows the fine tuning of precision and recall measures - raising generally increase precision at a cost of lower recall.
• K-means is a simple clustering-based classifier that allows for many classes. It uses a number of mean points to iteratively classify data points based on the distance from the mean points.
• Negative selection algorithm (NSA) is a one class classification method. it uses a self and non-self set to distinguish between normal and abnormal data points. It creates the self region by learning normal behaviour and labelling any data points that lie outside this region as non-self.

...Time-series prediction for resource utilisation
• There are multiple approaches to the problem of misallocation in cloud/edge networks, I will compare some for my context.
• ARIMA is a time series model which can take historical workload data to predict future workloads in order to more efficiently provision resources.
• LSTM-RNN is a neural network which can be used for workload prediction. It also takes historical workload as input but captures long-term patterns and handles non-linear workloads, enabling prediction on more complex workloads.
• Autopilot is an autoscaling system used in production which can take advantage of models such as LSTM-RNN to automatically allocate resources on future workloads. The aim of Autopilot is to reduce slack (the difference between resources allocated and resources needed). It does this by automatically adjusting horizontal scaling (number of tasks in a job) and vertical scaling (CPU and memory limits for each task).
• An alternative neural network approach is Evolutionary NN (ENN) which combines artificial NN with evolutionary algorithms which automatically optimise the NN structure by mutation, crossover, and fitness evaluation. Performance is measured using MAE and RMSE, usually with strong results at high computational cost.
• Together, these papers demonstrate how accurate workload and resource prediction—using statistical models, neural networks, and deep learning—enables automated autoscaling systems that improve efficiency and reliability in large cloud data centers.

...Intergration with orchestration
• Reinforcement learning schedulers use monitoring, learning of workload patterns, and trial-and-error interactions to adapt their scheduling strategy. Improvement is achieved through maximizing rewards, in turn, minimizing pod latency, balancing loads efficiently, and optimizing throughput. RL is able to achieve this due to its flexible nature allowing for it to evolve with workloads.
• Autoscaling is a trusted method for increase the performance and availability of cloud services. Autoscaling on kubernetes appears in the form of managing the number of instances and the resources allocated to them. Enhancing autoscaling with a prediction model can help reduce resource cost and maximize quality of service.
• Anomaly-triggered rescheduling is an advanced proactive maintenance strategy that moves running pods to better-suited nodes when specific anomalies such as performance bottlenecks, node resource exhaustion, or security threats are detected. This is essential since relocating nodes before failure is more effective than reacting when they do.

\subsection{Limitations of existing methods}

• The issue with static schedulers is that they ignore runtime state meaning they lack the context to make intelligent decisions which is necessary in dynamic edge environments.
• Reactive heuristics attempt to remedy this by using state to match predefined criteria to make decisions. Although better than static schedulers this still leads to problems with noise and lagging and fails to perform with dynamic workloads due to lack of anticipation.
• Anomaly detection is an improvement on threshold/rule based heuristics by more effectively detecting spikes, especially in dynamic environments. However, fails to act on the detection to make decisions.
• Prediction is more effective than reactive heuristics because it uses a proactive approach to forecast load to make better decisions.
• What is missing in contemporary methods is a lightweight system that combines both anomaly detection and prediction to inform scheduling.
